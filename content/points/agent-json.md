# 结构化输出：JSON Schema 与强校验

## 目标
把不确定性输出变成“可解析、可校验、可执行”的结构。

## 1) 为什么必须用 schema
如果只靠自然语言：

- 格式会漂
- 字段会漏
- 幻觉会扩散到下游

Schema + 校验的组合可以把错误挡在“模型输出 -> 业务执行”的边界上。

## 2) 典型字段
以精准推课为例：

- user_id
- course_ids（数组）
- reason（引用证据）

## 3) 失败策略
- 校验失败 -> 把错误信息反馈给模型 -> 重试
- 超过重试次数 -> 降级到默认课表

## 自检清单
- 你的系统是否能在进入数据库写入前保证字段合法？
