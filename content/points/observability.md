# 可观测性：日志/指标/链路追踪在 LLM 系统里怎么做

## 你要追踪什么
一次回答背后至少包含：

- 输入摘要
- 检索查询与 TopK
- rerank 结果
- 模型请求/响应（摘要）
- 工具调用链与耗时
- token 与成本

## 1) 日志
建议结构化日志：

- request_id
- user_id/tenant_id
- stage（retrieve/rerank/generate/tool）

## 2) 指标
- p50/p95 延迟
- 失败率
- token 消耗

## 3) 链路追踪
把“检索 -> 生成 -> 工具”串起来，才能定位问题。
