# 预训练：从数据到目标函数

## 预训练在做什么
预训练阶段把模型变成一个强大的“语言压缩器”：

- 学到语法与事实的统计规律
- 学到世界知识的共现结构
- 学到推理模式（一定程度上）

## 1) 数据：决定上限
你会看到的关键工程动作：

- 去重：避免模型背诵与过拟合
- 质量过滤：低质量文本会拖垮训练
- 领域配比：通用/垂直数据比例决定模型偏好

对你们的涉农政策库来说：

- 需要保留“条款结构、编号、引用链”
- 需要保留“专有名词与地名”

## 2) 目标：next-token prediction
最常见：自回归语言模型目标。

- 输入前缀
- 预测下一个 token

训练在每个 token 位置产生损失并求和/平均。

## 3) 序列长度与训练策略
序列越长：

- 训练更贵
- 但能学到更长程依赖

现实里会组合：

- 多种长度的混合
- curriculum（先短后长）

## 4) 数据分布与模型行为
如果数据里：

- 事实性内容多，模型更像百科
- 指令数据多，模型更像助手

因此预训练只是基础，后续需要对齐与指令微调。

## 与项目落地的连接
你们大概率不会从零预训练，但需要理解：

- 预训练模型会“倾向于流畅”而非“倾向于正确”
- RAG/引用是让它正确的重要机制
