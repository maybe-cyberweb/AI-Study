# Harness Engineering：Agent = Model + Harness

## 为什么需要 Harness
模型天然是无状态、不可控的文本生成器。生产系统需要：

- 状态与记忆
- 工具调用
- 校验与纠错
- 预算与熔断

这些都在 Harness 层完成。

## 1) 典型运行环（While Loop）
一轮 Agent 交互可以抽象为：

- 输入（用户问题 + 上下文）
- 计划（模型输出下一步）
- 行动（调用工具/检索）
- 观测（工具结果）
- 校验（传感器检查）
- 终止（输出给用户）

## 2) 传感器（Sensors）
传感器是确定性代码：

- 校验课程 ID 是否存在
- 校验引用段落是否能命中
- 校验 JSON schema 是否通过

## 3) 预算与熔断
必须限制：

- 单次对话工具调用次数
- token 消耗
- 执行时间

否则会出现烧钱死循环。

## 与项目落地的连接
你们的测评与推课都必须“可解释、可追溯、可回放”，Harness 层是底座。
