# GraphRAG：用结构化关系提升复杂问题的可解释性

## 适用场景（为什么你们需要它）
政策与农业业务的问题经常是“链式影响”：
- 某政策 → 影响哪些主体（村/合作社/农户）？
- 某培训计划 → 关联哪些岗位能力与课程？
- 某补贴条件 → 与哪些材料、审批流程相关？

传统向量 RAG 更像“找相似段落”，容易丢掉 **实体关系与跨文档关联**。

## 1) GraphRAG 的思路
把知识变成“图”：
- 节点：实体（政策、条款、主体、材料、岗位、课程）
- 边：关系（适用、依赖、包含、冲突、引用）

回答问题时不只做相似度检索，还会：
- 在图上找到相关的子图（community / neighborhood）
- 对子图进行摘要（community summary）
- 输出“证据链”（路径）与引用

## 2) 工程落地步骤（最小可用）
1) 文档解析：保留层级与引用定位（章/节/条/附件）。  
2) 实体抽取：规则 + 模型混合（先用规则抽政策名/编号，再让模型补全）。  
3) 图存储：选择图数据库或“图结构 + 向量索引”的混合存储。  
4) 查询：先定位实体，再扩展邻域，最后生成答案并附引用。  

## 3) 与 Hybrid RAG 的关系
- Hybrid RAG：解决“召回稳定”，尤其是专有名词/编号。
- GraphRAG：解决“关系解释”，尤其是跨文档、跨条款推理。

最佳实践：Hybrid 先召回候选文档 → Graph 再组织关系与证据链。

## 自检清单
- 我能列出你们场景中最关键的实体与关系吗（政策、课程、岗位、能力维度）？
- 我能让答案输出引用 + 关系路径（证据链）吗？
- 我能把图构建做成增量更新，而不是每次全量重建吗？
