# 上下文工程：Write / Select / Compress / Isolate

## 为什么“上下文工程”比 prompt 更关键
模型的行为强依赖输入上下文。生产系统的问题往往不是“提示词不够长”，而是：

- 上下文塞太多导致注意力迷失
- 多用户混在一起导致泄露
- 长对话导致规则失效

## 1) Write：写入长效记忆
写入的不只是对话，还包括：

- 用户偏好
- 工具调用日志
- 关键结论与证据

## 2) Select：只选最相关
RAG 的核心是选择：

- TopK 不是越大越好
- 需要 metadata 过滤（地域/年份/政策类型）

## 3) Compress：摘要压缩
当对话变长：

- 中间历史用摘要替代
- 保留系统提示与最新对话

## 4) Isolate：隔离
隔离必须是“物理隔离/权限隔离”，不是“让模型记住别泄露”。

## 自检清单
- 你能说清楚“该写入什么”和“不能写入什么”吗？
- 你能解释为什么 TopK 越大不一定越好么？
