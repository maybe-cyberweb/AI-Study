# LoRA/QLoRA：低成本定制行业模型

## 你为什么需要 LoRA
你们的场景（政策解读、推课、测评）需要风格稳定与结构化输出，但不一定需要“重写世界知识”。

LoRA 的价值：

- 参数高效
- 训练成本低
- 可快速试错

## 1) LoRA 的形式
把权重更新写成低秩：

- ΔW = A B

rank r 越大：

- 表达更强
- 成本更高

## 2) QLoRA
QLoRA 在量化权重上训练 LoRA：

- 显存更省
- 易在单卡上跑

代价：

- 数值更敏感
- 需要更谨慎的超参

## 3) 什么时候不该 LoRA
- 你需要大量事实更新（应该用 RAG）
- 你需要严格可追溯引用（应该用 grounding 与 schema 校验）

## 自检清单
- 你能说清楚 LoRA 适合“风格/格式”还是“知识更新”吗？
- 你知道 rank 变化带来的工程代价吗？
