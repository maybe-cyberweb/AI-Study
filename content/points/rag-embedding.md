# Embedding：相似度检索背后的表示学习

## 你要掌握的三件事
- 向量表示的语义含义
- 相似度度量（cosine / dot）
- 召回与噪声的平衡

## 1) Embedding 是“把句子变成坐标”
文本经过编码器得到向量：

- 语义相近 -> 向量更近

因此你可以用“近邻检索”找到相关段落。

## 2) 维度与域
维度越大：表达更强，但存储与计算更贵。

领域差异：

- 通用 embedding 对政策条款不一定最优
- 可考虑领域微调或 rerank 修正

## 3) 工程建议
- 统一归一化策略（是否 L2 normalize）
- 保存 metadata（来源、条款号、发布时间）用于过滤

- 你能说清楚“向量近”意味着什么吗？
- 你知道 metadata 过滤为什么对涉农政策库很关键吗？
