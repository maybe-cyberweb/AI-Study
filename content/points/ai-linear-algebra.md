# 线性代数在 LLM 里的真实作用

## 你学线代不是为了做题
在大模型里，线性代数不是“背景知识”，而是模型计算的本体：

- embedding 是查表 + 线性映射
- attention 是大量矩阵乘
- MLP 是两次线性层
- LoRA 是低秩矩阵分解

你需要掌握的重点是：**向量、矩阵、投影、低秩近似**。

## 1) 向量：表示与相似度
当我们说“语义相近”，通常意味着向量空间距离更近。

- 余弦相似度：cos(a,b) = (a·b) / (||a|| ||b||)
- 点积越大，方向越相近

Embedding 检索就是在做“相似度最大化”。

## 2) 矩阵乘：把一个空间变换到另一个空间
线性层：y = xW

- W 的列空间决定了输出能表达的方向
- 多层叠加 + 非线性（激活函数）带来表达能力

在 Transformer 里：

- Q = XWq, K = XWk, V = XWv
- 每个 W 都是在做“把 token 表示投影到不同子空间”

## 3) 投影：注意力其实是在选信息
注意力权重：softmax(QKᵀ)

- QKᵀ 计算相似度
- softmax 把相似度变成概率
- 再乘 V 做加权求和

这就是“按相关性聚合信息”。

## 4) 低秩近似：LoRA 的直觉
LoRA 把一个大矩阵的增量 ΔW 写成：

- ΔW = A B

其中 A、B 的 rank 很小。

直觉：

- 行业微调往往只需要在少量方向上调整
- 低秩更新足以表达“风格/知识偏置”的改变

## 5) 你在工程里会用到的“线代动作”
1) 选 embedding 维度：维度过小表示不够，过大增加存储与计算。
2) 向量归一化：决定余弦相似度是否稳定。
3) 低秩/量化：决定推理成本。

## 6) 与项目落地的连接
你们的政策库与干部档案会进入 RAG：

- 文本 -> embedding 向量
- ANN 索引近似最近邻

如果你能解释“为什么相似度有效”，你就更容易调好：

- 切片策略
- 召回 TopK
- 重排策略
