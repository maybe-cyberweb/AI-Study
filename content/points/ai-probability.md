# 概率、似然、交叉熵：从分类到语言模型

## 你要解决的核心问题
大模型训练看起来很玄学，但其目标函数非常朴素：让模型给“正确答案”的概率尽可能大。要理解它，你需要三件事：

- 概率（probability）
- 似然（likelihood）
- 交叉熵（cross entropy）

## 1) 从“分类”到“下一 token 预测”
监督分类：给定输入 x，预测标签 y。

- 模型输出一组 logits，经 softmax 变成概率分布 p(y|x)
- 训练目标：最大化正确标签的概率

语言模型：给定前缀 token 序列 x1..xt，预测下一个 token xt+1。

- 模型输出 vocab 大小的 logits
- softmax 得到 p(xt+1 | x1..xt)
- 对整段文本求和：最大化每个位置“真实下一个 token”的概率

一句话：**LLM 训练就是一个超大规模的多分类问题**。

## 2) 似然与负对数似然（NLL）
如果训练集里有 N 个样本（或 N 个 token 位置），我们希望模型对真实标签给高概率。

最大化似然：

- L = Π p(y_i | x_i)

为了数值稳定，取对数：

- log L = Σ log p(y_i | x_i)

通常我们最小化负对数似然：

- NLL = - Σ log p(y_i | x_i)

## 3) 交叉熵与“为什么用它”
对于 one-hot 标签（真实类别是 y*），交叉熵就是：

- CE = - log p(y* | x)

这与 NLL 等价。

直觉：

- 预测越自信且正确（p 越接近 1），损失越小
- 预测越自信但错误（p 很小），损失会急剧增大

这就是为什么交叉熵非常适合作为训练目标。

## 4) 语言模型中的 CE：per-token 与 perplexity
语言模型通常报告：

- 平均 token 交叉熵：CE_avg
- 困惑度 perplexity：PPL = exp(CE_avg)

PPL 的直觉：平均而言，模型每一步相当于在多少个候选项里“猜”。越小越好。

## 5) 工程视角：你在训练里会真的碰到什么
你会遇到：

- **label shift**：输入是前缀，标签是下一个 token
- **masking**：对 padding 位置不计入损失
- **混合精度**：loss scaling 防止梯度下溢

典型伪代码（概念级）：

```text
logits = model(tokens[:, :-1])
labels = tokens[:, 1:]
loss = cross_entropy(logits, labels, ignore_index=PAD)
```

## 6) 和你的项目的关系（农业干部教育）
你们的 AI 知识库、测评、推课、伴读本质都是“条件生成”：

- 输入：用户问题/干部档案/政策条款
- 输出：一段结构化或半结构化的文本

理解 CE/NLL 能帮助你做两件事：

1) 解释“为什么模型会胡说”：模型只是在最大化概率，并不天然知道事实
2) 解释“为什么 RAG 能缓解幻觉”：把事实塞到上下文里，让正确答案概率变大
