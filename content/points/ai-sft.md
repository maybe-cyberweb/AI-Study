# 指令微调（SFT）：让模型会“按要求回答”

## SFT 的目标
把“会写字的模型”变成“会按规则办事的助手”。

常见形式：

- system / user / assistant 三段式
- 对 assistant 部分计算损失

## 1) 数据格式
一个高质量 SFT 样本通常包含：

- 清晰的任务指令
- 明确的约束（格式、风格、边界）
- 合格的参考答案（结构清晰、可执行）

## 2) 规则与红线
SFT 可以注入：

- 政策红线与合规话术
- 引用要求（必须给出处）
- 工具调用协议（必须输出 JSON）

但要记住：

- **提示词不是安全**
- 仍需要 Harness/权限校验

## 3) 常见坑
- 数据噪声：会把模型往坏方向拉
- 风格漂移：加入太多“模板化废话”会影响真实可用性
- 过拟合：小数据大步长很容易

## 与项目落地的连接
你们可以用 SFT 做：

- 伴读助手的风格统一
- 政策解读的结构化输出
