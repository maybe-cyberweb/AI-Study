## 阶段一：筑基与全栈协同
建立统一的技术语境，围绕大模型进行协议对接。

### 1.1 前端：从静态页面到 AI 实时流交互
- **SSE (Server-Sent Events)**：实现大模型逐字吐出的流式渲染。
- **动态 JSON 渲染**：利用 `JSON.parse` 实时拦截格式化数据，动态驱动 UI 组件。
- **可视化框架**：集成 `React Flow` 展现复杂的教育调度逻辑。

### 1.2 后端：高性能 API 与状态保持
- **Session 状态管理**：基于 Redis 缓存会话历史，赋予模型上下文记忆。
- **高吞吐传输层**：使用 Go (Gin/Fiber) 或 Python (FastAPI) 构建无损 SSE 转发通道。

## 阶段二：上下文工程与数据治理
从“盲目写 Prompt”跨越到真正的“上下文工程”。

### 2.1 上下文工程四大策略
1. **Write (写入)**：长效记忆数据库（PostgreSQL）存储原始日志。
2. **Select (选择)**：利用 RAG 机制过滤 Top-K 相关知识，拒绝过载。
3. **Compress (压缩)**：利用小模型对历史会话进行摘要压缩，节省 Token。
4. **Isolate (隔离)**：物理隔离不同角色、不同干部的会话上下文。

### 2.2 进阶 RAG 体系
- **语义切片 (Semantic Chunking)**：基于余弦相似度在语义断崖处切片，保证知识连贯。
- **混合检索 (Hybrid Search)**：结合向量相似度与 BM25 关键词，通过 RRF 算法合并排序。
- **最新趋势：GraphRAG**：针对农业政策中的复杂实体关系，构建知识图谱以提升宏观问题的回答能力。

## 阶段三：驾驭工程 (Harness Engineering)
**核心公式：Agent = Model + Harness**

### 3.1 Harness Layer 的核心职能
- **大脑 vs 发动机**：Model 负责思考（大脑），Harness 负责运行环境（Runtime/发动机）。
- **反馈闭环 (Feedback Loop)**：
    - **前馈控制**：系统提示词 (System Prompt) 设定政策红线。
    - **传感器反馈**：自研校验代码（传感器）验证模型输出的业务数据（如课程编号）。
- **最新标准：MCP (Model Context Protocol)**：
    - 解耦数据源与模型，通过标准 SDK 编写 MCP Server 连接本地档案与题库。

## 阶段四：高级算法与量化评估
拒绝“凭感觉调优”，引入自动化评估体系。

### 4.1 Ragas 评估框架
- **Faithfulness (忠实度)**：通过 NLI 算法判断答案是否源于检索上下文，消除幻觉。
- **Context Precision (精确度)**：评估检索出的段落排序是否合理。
- **Context Recall (召回率)**：评估检索系统是否漏掉关键政策。

## 阶段五：实战落地避坑指南
- **困难一：幻觉级联** -> 解法：JSON Schema 强约束 + 步进拦截。
- **困难二：上下文退化** -> 解法：滑动窗口 + 核心上下文锚定 (Anchor)。
- **困难三：Vibe Coding 风险** -> 解法：人类 Review 核心逻辑 + 自动化安全扫描。
